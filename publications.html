Применение структурного подхода к проектированию программного обеспечения

В данной работе рассматривается процесс создания программного продукта, представляющего собой имитацию процесса пропуска в учреждение (банк), предназначенного для организации контролируемого доступа на объект (банк) сотрудников и посетителей. Требования к программе предусматривают выполнение следующих функций:

  • считывание пропусков;
  • хранение информации о сотрудниках и постоянных посетителей учреждения;
  • получение информации о текущем состоянии пропуска;
  • автоматическое распознавание данных с документов;
  • выдача процента зафиксированных посторонних;
  • уведомление о результате допуска/не допуска;
  • выполнения начальной настройки системы.

Можно выделить несколько особенностей построения модели ПО:

  1. Необходимость наличия базы данных;
  2. Участие в моделируемом процессе специального считывающего устройства.

Все объекты, связи между ними, а также их состояния, используемые в построении модели образуют предметную область. Набор моделей предметной области позволяет описать деятельность участников рассматриваемого процесса (вахтера, охранника, дежурного и т.п).

Что такое структурный подход? Структурный, или иначе, функционально-модульный подход – это подход, при котором структура системы иерархически упорядочена по функциям и передаче информации между отдельными функциональными элементами. То есть при проектировании сложного ПО прежде всего необходимо определить структурные компоненты и связи между ними.

При разработке ПО с использованием структурного подхода строятся модели, описывающие функциональную структуру системы и отношения между данными. Наиболее распространенными являются следующие модели:

  • DFD (Data Flow Diagrams) – диаграммы потоков данных;
  • SADT (Structured Analysis and Design Technique – метод структурного анализа и проектирования) – модели и соответствующие функциональные диаграммы;
  • ERD (Entity-Relationship Diagrams) – диаграммы «сущность-связь»;
  • STD ( State Transition Diagrams) – диаграммы переходов состояний.

Более подробно в этой работе рассмотрен сценарий программы пропуска в учреждение или методология IDEF3, а также диаграмма потоков данных. Структурное моделирование IDEF3 позволяет выделить основные функции предметной области. Информационный аспект взаимодействия объектов отображается с помощью диаграмм потоков данных с основными компонентами - внешние сущности; процессы; накопители данных или хранилище и потоки данных.

Необходимость диаграммы потоков данных обусловлена обязательным этапом выявления основной информации, обеспечивающей успешную работу этой модели ПО. Изначально строится контекстная диаграмма, затем диаграмма первого уровня, в которой есть два процесса(«работа» и «оформление»). На диаграмме второго уровня детализирован процесс «работа».

Контекстная диаграмма показывает, как система будет взаимодействовать с пользователем (т.е. с посетителем) и другими внешними системами: бюро пропусков и считывающее устройство – проксимити-считыватель. Proximity – это дистанционный электронный пропуск. Proximity используются для бесконтактной идентификации его владельца при определении прав доступа на объект, защищенный системой контроля.

Для реализации программы использовалась среда программирования Visual Studio и язык программирования Visual Basic.NET.

После запуска программы с помощью генератора случайных чисел имитируется появление посетителя учреждения (работник этого учреждения или обычный клиент).

Каждый результат работы программы записывается в файл, на основе которой формируется необходимая статистика. Полученная информация о результатах допуска позволит спрогнозировать, например, количество посетителей в следующий месяц или в следующий квартал; или поможет скорректировать процесс замены пропусков с просроченных на новые.

Применение структурного подхода позволило представить различные аспекты предметной области и функции программного продукта, а также выявить все особенности моделируемых процессов.

Оценка эффективности инвестиционного проекта на основе имитационного моделирования

Целью работы является разработка подхода к оценке эффективности инвестиционного проекта с учетом рисков на основе имитационного моделирования, предметом исследования выступает сам процесс реализации инвестиционного проекта.

В современных условиях успешная реализация инвестиционных проектов возможна только на основе применения теории управления проектами. Важным этапом управления проектами является этап оценки эффективности инвестиционного проекта с обязательным учетом риска.

На этапе разработки проекта на основе применения количественных методов оценки риска оценивается эффективность и уровень риска. Наиболее полная оценка эффективности инвестиционного проекта с учетом уровня риска может быть получена на основе совместного применения методов анализа чувствительности и имитационного моделирования. Не во всех случаях для этой цели может быть применен программный продукт Project Expert.

Для оценки эффективности инвестиционного проекта на основе имитационного моделирования предлагается использовать программный продукт IThink. Логическая схема имитационного моделирования приведена на рисунке 1.
